# Code of Conduct

## Our Standard

This project aims to keep collaboration respectful, direct, and constructive.

Examples of expected behavior:

- discuss ideas in good faith
- give specific technical feedback
- respect different experience levels
- accept correction without hostility
- focus criticism on the work, not the person

Examples of unacceptable behavior:

- harassment or personal attacks
- insulting or demeaning language
- bad-faith disruption
- publishing private information without permission
- discriminatory behavior of any kind

## Enforcement

Project maintainers may remove, edit, or reject comments, issues, commits, code, and other contributions that violate this Code of Conduct.

## Scope

This Code of Conduct applies within project spaces and in public spaces when someone is representing the project.

## Reporting

If you experience or witness unacceptable behavior, contact the maintainer through the repository's issue tracker or listed contact channel.

## Notes

This is a lightweight project code of conduct intended for small open source collaboration. It can be replaced later with a fuller standard such as the Contributor Covenant if needed.
